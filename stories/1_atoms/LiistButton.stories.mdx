# `<liist-bttn>`

Main button component.

## Properties

| Property       | Description | Default |
| -------------- | ----------- | ------- |
| `text`         | text content of the button. | - |
| `thin`         | Boolean variable, set font-weight normal instead of bold. | false |
| `theme`        | theme of the button, see [LiistColors documentation](?path=/docs/atoms-liistcolors--page). **IMPORTANT**: overrides `color` and `bgColor`. | `viiolet80` |
| `color`        | text color of button. Only applied when `theme` is set to `custom`. | - |
| `bgColor`      | background color of the button. Only applied when `theme` is set to `custom`. | - |
| `icon`         | Name of the icon. For example `pin`, `ufo`, `filter`, ... | `undefined` |
| `iconPos`      | position of the icon. `left` or `right`. | `left` |
| `width`        | width set in percentage or pixels. For example `250px` or `100%` | `100%` |
| `url`          | url as string. If no url is set, button is not clickable! | - |
| `openInNewTab` | Boolean. Opens url in new tab. Only applied when url is set. | - |

## Javascript OnClick Event
Instead of using the `url` property, you can also perform a javascript action onclick by adding an onclick attribute in the root component. <br></br> **Important**: you have to manually set the cursor to pointer (for now, can be fixed later).

```html
<liist-bttn
  onclick="alert('Hello, Friend!')"
  style="cursor: pointer"
  text="Say Hi"
></liist-bttn>
```

  ></liist-bttn>
</Preview>
